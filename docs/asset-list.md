# 素材清单（Asset List）

> 状态：v1（2026-07-17）。从 9 份设计文档收拢的全部美术/音频/字体需求，**后续素材管理以本文件为准**：新增素材先登记，完成后更新状态列。
> 设计依据：MVP 用占位美术（几何图形/色块）先验证手感（game-design MVP 切片），所以每项素材有两个完成度——**占位**（引擎内色块/系统音，能玩）和**正式**（美术资产）。
> 存放路径遵循 CLAUDE.md：图片 `assets/sprites/`、音乐 `assets/audio/music/`、音效 `assets/audio/sfx/`、字体 `assets/fonts/`；文件名 snake_case。

## 状态标记

| 标记 | 含义 |
|---|---|
| ⬜ | 未开始 |
| 🟨 | 占位已做（色块/几何/系统音，MVP 可用） |
| ✅ | 正式素材完成 |
| ⏸ | 非 MVP，后续版本 |

**MVP 完成标准 = 所有非 ⏸ 行到达 🟨**；正式素材（✅）在手感验证后分批替换。

## 管理约定

1. **先登记后制作**：设计新增素材需求时，先在本表加行（状态 ⬜），再进生产
2. **占位不欠账**：MVP 期任何素材先做 🟨 占位（色块/几何/免费音效），全表 🟨 即可开始试玩验证
3. **正式替换分批走**：手感验证通过的系统优先替换正式素材（✅），替换顺序：玩家/敌人 → 武器特效 → UI → 场景 → 音频
4. **来源标注**：外购/开源素材在状态列后补记来源与许可证（如 `✅ (Kenney, CC0)`），自制标 `✅ (自制)`
5. 规格建议列以 docs/tech-design.md §1 的像素密度基准为准（32px 角色基准、1920×1080 视口）；相机 zoom 终值已实测拍板 1.0（M0 验收）
