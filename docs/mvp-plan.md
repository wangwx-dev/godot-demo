# MVP 实施计划

> 状态：v1（2026-07-17）。依赖顺序展开为里程碑，每个里程碑挂设计文档现成的验收问题——做完一步有明确的"算不算成"。
> 范围以 game-design.md v4.1 的 MVP 切片为准；素材完成度跟踪见 docs/asset-list.md（全表 🟨 = 可试玩）；技术方案见 docs/tech-design.md。

## 里程碑总表

| # | 里程碑 | 内容 | 验收（来源文档） |
|---|---|---|---|
| M0 | **工程骨架** | 相机跟随+zoom 实测拍板、EventBus/RunRng/RunState/Debug 四 autoload、输入映射、Resource 类定义、调试角标 | 色块玩家在 2560×2560 空图里跑，一屏体感 OK，种子参数生效 |
| M1 | **敌人 + 自动武器** | 普通尸/奔跑者/臃肿者（状态机基类+分离力群移+对象池）、球棒/手枪自动索敌、经验球/金币掉落、接触伤害/受击无敌/死亡 | 奔跑者出现时玩家是否明显紧张？臃肿者是否让近战改打法？球棒和手枪是否明显是两种游戏？（enemy/weapon） |

| M4 | **资源点 + 经济** | 四种资源点+驻留（刷入减半）+绷带散布、8 格背包+满包替换、价值物三档、货币 | 满包开箱时玩家会犹豫吗？会为省 20 金治疗费冒险吗？（economy） |

| M6 | **载具 + 路线 + 天数** | 载具出口（摆放约束+目的地牌）、路线生成（权重+去重+保底）、天数推进、选路界面、休整图（治疗/重随/信号弹）、词缀精英+精英图 | 两辆载具目的地不同时会纠结吗？会在"还剩几天"层面做规划吗？两局精英感觉不同吗？（mapgen/game/enemy） |
| M7 | **简版商店 + 结算** | 商店图（自动兑换播报+2 格货架）、正式警戒条/HUD 四簇、死亡/撤离结算画面 | 背包快满时会主动选商店路线吗？警戒条不看说明能懂吗？（economy/ui） |
| M8 | **总攻占位 + 全循环** | 总攻图（三词缀超级精英+崩溃级涌潮）、提前发起、Boss 血条、完整一局串通 | 会在"再搜一个箱子"和"赶紧走"之间纠结吗？——核心张力总验收（game-design） |

## 执行约定

- **顺序即依赖**：M1 的敌人是 M2 刷怪的弹药，M4 的经济是 M6 休整服务的燃料——不跳步
- **每个里程碑一次试玩**：回答验收问题，顺手校准该层数值（.tres 改数不动代码）；答不上来就停下调，不带病推进
- 每里程碑一个 commit 系列，完成时更新本表状态（表格加 ✅ 前缀）
- M2 起每次试玩顺带盯性能角标：同屏 50 只 + 分离力在 GL Compatibility 下的帧率——超预算优先降分离力计算频率
- 素材：进入 Mx 前把该里程碑涉及的 asset-list 行推到 🟨（占位不欠账）
