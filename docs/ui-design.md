# 信息与 UI 设计

> 状态：v1（2026-07-17）。基调：合并警戒条、常驻迷你图、无数字极简反馈。带 ❓ 的是待定项。
> 本块收拢各系统甩来的呈现待办：Heat/死线分层（pressure #6）、强化图标层数（upgrade #5）、词缀图标（enemy）、天数/路线界面（game-design v4）。**布局为初版，实装后按体感迭代。**

## 设计原则

1. **压力清晰是支柱**（压柱 2）：玩家永远知道自己为什么死——所有威胁信息（血量、警戒、涌潮方向）优先级最高，永远可见、永远醒目
2. **决策资源必须常驻**：天数、货币、背包余量是每个决策的输入，藏进菜单=决策变盲选（压柱 1）
3. **画面为战场服务**：中央区域不放任何常驻 UI；数字能不给就不给（无伤害飘字，敌人不显血条），状态用形态/颜色表达——"轮廓即身份"延伸为"形态即状态"

## 战斗图 HUD 布局

```
┌─────────────────────────────────────────────┐
│ [血条+血量数字]                [尸潮警戒条]  │
│ [经验条(细)]                     [迷你图]    │
│ [主武器|副武器CD]                            │
│                                              │
│                （战场，无常驻UI）            │
│                                              │
│ [技能CD图标]              [第X天|金币|背包]  │
└─────────────────────────────────────────────┘
```

- **左上（生存簇）**：血条（红，带数字 87/100——HP 是胆量资源，要精确到能算账"再挨 3 下"）；下挂细经验条（升级瞬间闪光即可，无数字）；主/副武器图标+副武器冷却扇形
- **右上（威胁簇）**：尸潮警戒条 + 迷你图，威胁与空间信息放同一视线角
- **左下**：角色技能冷却图标（翻滚），大而醒目——保命键的可用性必须余光可读
- **右下（资源簇）**：第 X/10 天、金币数、背包 5/8——三个决策资源一行排开
- 拾取反馈：金币/经验被吸附时的音效+轨迹即可，无飘字；捡到价值物弹小横幅（"军用物资 +60"，1.5s 淡出）❓唯一允许的数字飘播，试玩定去留

## 尸潮警戒条（Heat + 死线合并呈现）

一条右上角横条，**填充进度 = 死线倒计时（5:00 → 满）**，**颜色/状态 = 当前 Heat**：

| Heat | 颜色 | 状态 |
|---|---|---|
| 0~3 | 绿 | 平稳 |
| 4~7 | 黄 | 缓慢脉动 |
| 8+ | 红 | 快速脉动 |
| 死线后（崩溃） | 深红 | 条满溢出、边框闪烁 + 持续心跳音 |

- 最后 60 秒：条身开始闪烁 + 心跳音效渐强；最后 10 秒中央短暂弹出大数字逐秒报数（允许的第二处数字——执行精度需要）
- 涌潮预警（3s）：**屏幕对应边缘方向性红光**（信息在世界里，不在 UI 里）+ 警戒条短促闪两下
- 休整图：警戒条隐藏（无压力空间连表都不给，安全感做足）
- 设计意图：填充管"还剩多久"（趋势可读），颜色管"现在多凶"（状态可读）——一个元件两层信息，牺牲的精确秒数由末段数字补回

## 迷你图（常驻，右上）

- 圆形❓/方形小图，显示：已探索区域轮廓（半透明）、未探索全黑、玩家箭头、**已发现的载具图标（含目的地类型色）**、已发现未开的资源点、精英图标（发现后）
- 不显示：普通敌人、迷雾外任何信息
- 载具图标是迷你图的核心价值：找到过的出口不用记路——"决定走的时候知道往哪跑"是撤离执行的信息底座
- ❓ 涌潮来向是否在迷你图边缘也标（与屏幕红光冗余）——试玩定

## 战斗反馈（无数字极简）

- 命中：敌人闪白 1 帧 + 受击音；击杀：溶解/碎裂 + 掉落物弹出
- 敌人不显血条；**例外：精英显血条+词缀图标**（头顶，沿 enemy-design），Boss 同理
- 玩家受击：屏幕边缘红晕 + 短震屏 + 0.5s 无敌闪烁（已有锚点）；血量 <30%：血条脉动 + 屏幕边缘常驻淡红晕——"该去休整了"不用文字说
- 构筑收益验证的替代通道（没有飘字时玩家怎么感知变强）：击杀速度本身 + 升级选卡界面显示当前武器面板数值❓（DPS/间隔/范围，给规划型玩家的账本入口）

## 选路界面（离图时）

- 乘载具出发 → 结算横幅（本图收获：物资 X 件（未变现）/金币 +Y/击杀 Z——3s 可跳过）→ **日历式路线界面**：顶部"第 X 天/共 10 天"进度轴，中央 2~3 张候选图卡（类型图标+名称，如"战斗·废弃超市区"），已定内容只有类型——奖励倾向预览是第二版❓
- 该界面同时是喘息点：显示当前构筑摘要（武器等级/已拿强化图标网格，强化图标+层数点：upgrade #5 在此闭环）、背包清单
- 信号弹（提前总攻）确认弹窗：显示"剩余 X 天将放弃"——放弃的天数要摆在眼前再让玩家按

## 升级三选一界面（暂停）

- 三张卡：稀有度边框色（白/蓝/紫）、图标、名称、一句话效果、（已有层数 ●●○ 显示）
- 底部：重随按钮（显示当前价格 10/20/40…金 + 持有金币）、跳过按钮（+15 金）
- 紫卡（规则改写）出现时卡片特殊动效❓——"这是大事"的仪式感，内容期定
- 武器专属强化标注所属武器小图标——防止误读为通用

## 商店图 UI

- 进图即弹兑换播报：物资清单逐行滚动折算（罐头 ×3 → 30 金…合计 +140 金）——变现的爽感要给足仪式
- 货架同休整图服务空间化（柜台交互）
- 背包簇兑换后归零刷新

## 休整图 UI

- 走近服务点（火堆=治疗、工作台=重随/扩容、载具=出发、信号弹=总攻）弹交互提示——服务空间化，不做集中菜单❓（试玩若嫌绕路改弹菜单）
- 常驻 HUD 保留资源簇与血条，隐藏警戒条/武器 CD

## 总攻图 HUD

- 警戒条替换为 **Boss 血条**（顶部居中）——总攻的时钟是 Boss 的命
- 其余同战斗图；无迷你图（小图无迷雾）

## MVP 切片（UI 部分）

- HUD 四簇（占位样式：色块+系统字体）
- 警戒条完整逻辑（填充+变色+末 60s 闪烁+末 10s 报数+崩溃态）
- 迷你图：探索轮廓+玩家+载具图标（资源点图标可后补）
- 选路界面简版：天数轴+2 候选卡+背包/构筑列表
- 三选一界面（含重随/跳过/层数点）
- 极简战斗反馈全套（闪白/溶解/红晕/低血脉动）
- 验收问题：第一次玩的人不看说明能否说出警戒条是什么？血量 30% 以下玩家是否明显变保守？选路界面停留时玩家在看什么（眼动=信息布局是否对）？
